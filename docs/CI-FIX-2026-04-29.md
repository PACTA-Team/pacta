# CI Fix: Resolución de Errores de Compilación

**Fecha**: 2026-04-29  
**PR**: #301 - feat(ai): replace Phi-3.5 with Qwen2.5-0.5B-Instruct  
**Problema**: CI fallando en todos los builds de la rama feature/minirag-hybrid-integration

---

## Errores Detectados en CI

### Error 1: Syntax Error — Missing package declaration
```
internal/handlers/ai.go:1:1: expected 'package', found 'import'
```

**Causa**: Durante la implementación masiva, el archivo `internal/handlers/ai.go` se truncó y perdió la línea `package handlers` inicial.

**Fix**: Agregar `package handlers` al inicio del archivo.

**Línea afectada**: Línea 1 de `internal/handlers/ai.go`

---

### Error 2: Import Cycle
```
imports github.com/PACTA-Team/pacta/internal/ai from indexer.go: import cycle not allowed
```

**Ciclo detectado**:
```
internal/ai/client.go (paquete "ai") 
    └─> importa "github.com/PACTA-Team/pacta/internal/ai/minirag"
        internal/ai/minirag/indexer.go (paquete "minirag")
            └─> importa "github.com/PACTA-Team/pacta/internal/ai"  ← CICLO
```

**Causa**: `minirag/indexer.go` importaba `internal/ai` pero no utilizaba ningún símbolo de ese paquete (verificado con `grep -n "ai\." indexer.go` → cero coincidencias).

**Decisión arquitectónica** (según design doc):
- `minirag` es un **subpaquete autónomo** (motor RAG local independiente)
- Debe mantenerse **sin dependencias** hacia el paquete padre `internal/ai`
- `internal/ai` puede importar `minirag` (relación jerárquica, no ciclo)

**Fix**: Eliminar la importación innecesaria de `internal/ai` en `internal/ai/minirag/indexer.go`.

**Líneas afectadas**: Líneas 10-11 de `internal/ai/minirag/indexer.go`

---
